# Cerveau Produit — Conseil constitutionnel — Lot 1

## Value MVP

---

## 1. Promesse principale

La promesse du futur site est de transformer un dispositif institutionnel dense en une expérience claire, progressive et fiable.

Le site doit permettre aux citoyens de comprendre les textes, décisions et contenus du Conseil constitutionnel, tout en permettant aux profils experts d’approfondir efficacement grâce à des outils adaptés.

### Promesse centrale

> **Comprendre d’abord, approfondir ensuite.**

Cette promesse repose sur trois piliers :

1. rendre les contenus institutionnels et décisionnels plus compréhensibles pour les citoyens ;
2. créer une expérience de lecture progressive, du niveau accessible au texte officiel ;
3. produire un référentiel UX/UI robuste, exploitable par le titulaire du lot 2 pour la réalisation technique.

---

## 2. Valeur pour les citoyens

Pour les citoyens, la valeur principale est la compréhension.

Le futur site doit permettre de :

- comprendre le rôle du Conseil constitutionnel ;
- comprendre le sens général d’une décision ;
- identifier pourquoi une décision est importante ;
- comprendre ce qu’elle change concrètement ;
- accéder au texte officiel sans être perdu ;
- poursuivre sa recherche grâce à des contenus liés.

Le citoyen ne doit pas être obligé de maîtriser le vocabulaire juridique pour commencer son parcours.

L’expérience doit donc proposer une première couche claire, courte et structurée, qui aide à entrer dans le sujet sans remplacer le texte officiel.

---

## 3. Valeur pour les experts

Pour les experts juridiques, la valeur principale est l’efficacité.

Le futur site doit permettre de :

- accéder rapidement au texte officiel ;
- retrouver les références utiles ;
- consulter les documents liés ;
- utiliser des filtres experts ;
- exploiter les tables analytiques ;
- accéder aux dispositions déclarées conformes ;
- comprendre le parcours procédural d’une décision ;
- repérer les décisions associées ;
- citer, partager ou exporter plus facilement les informations utiles.

L’expérience citoyenne ne doit pas dégrader les usages experts.

Le principe retenu est donc :

> **Citoyen en entrée, expert en profondeur.**

---

## 4. Valeur pour les médias

Pour les médias, la valeur principale est l’accès rapide à une information fiable, claire et exploitable.

Le site doit permettre aux journalistes de :

- comprendre rapidement le sujet d’une décision ;
- identifier son statut ;
- repérer les éléments clés ;
- accéder au texte officiel ;
- trouver les contenus liés ;
- partager ou citer correctement une décision.

Le site doit faciliter la lecture rapide, sans produire de raccourcis susceptibles d’altérer la rigueur juridique.

---

## 5. Valeur pour les enseignants et élèves

Les enseignants et élèves ne constituent pas le cœur du futur site principal.

Le site **“Découvrons notre Constitution”** conserve son autonomie pédagogique.

La valeur du futur site principal est donc d’orienter clairement ces publics vers les ressources adaptées, sans recréer une couche pédagogique lourde dans le site institutionnel.

Le site principal doit proposer :

- des liens visibles vers “Découvrons notre Constitution” ;
- des accès depuis la home ;
- des passerelles contextuelles depuis les pages liées à la Constitution ;
- une orientation claire pour les publics scolaires.

---

## 6. Preuve de valeur attendue

La valeur du projet doit être démontrable.

La preuve principale repose sur les tests utilisateurs.

Les tests doivent permettre de vérifier que :

- les citoyens comprennent mieux l’essentiel d’une décision ;
- les citoyens savent passer d’une explication simple au texte officiel ;
- les utilisateurs trouvent plus rapidement une décision ou une information ;
- les experts accèdent facilement aux outils professionnels ;
- les médias identifient rapidement les informations clés ;
- les utilisateurs utilisent les contenus liés pour approfondir leur recherche.

La réussite ne doit pas seulement être évaluée par la qualité graphique des interfaces, mais par l’amélioration mesurable de la compréhension, de la recherche et de l’accès aux contenus.

---

## 7. Hypothèses à valider

La démarche UX doit permettre de valider plusieurs hypothèses structurantes.

### Hypothèse 1 — Compréhension citoyenne

Les citoyens ne veulent pas seulement consulter une décision.

Ils veulent comprendre :

- ce qu’elle signifie ;
- pourquoi elle existe ;
- ce qu’elle change ;
- comment aller plus loin.

### Hypothèse 2 — Niveaux de lecture

Les modes de lecture facilitent l’appropriation des contenus complexes, à condition de rester simples, peu nombreux et clairement reliés au texte officiel.

### Hypothèse 3 — Acceptabilité expert

Les experts acceptent une interface plus accessible au grand public si les outils professionnels restent visibles, rapides et puissants.

### Hypothèse 4 — Navigation courte

Une navigation principale limitée à cinq entrées peut couvrir les principaux usages, à condition d’être testée par :

- card sorting ;
- tree testing ;
- confrontation aux parcours prioritaires.

### Hypothèse 5 — Approfondissement

Les utilisateurs sont prêts à approfondir leur recherche si le site leur propose :

- des liens contextuels pertinents ;
- des décisions associées ;
- des définitions utiles ;
- des accès au texte officiel ;
- des outils experts accessibles.

---

## 8. Risques principaux

### Risque 1 — Sous-estimer les besoins experts

Le principal risque est de produire une expérience trop orientée grand public, au détriment des usages professionnels.

Pour éviter ce risque, le site doit intégrer dès la conception :

- un accès expert clair ;
- une recherche avancée ;
- des filtres professionnels ;
- les références officielles ;
- les documents liés ;
- les outils juridiques attendus ;
- une lecture officielle toujours accessible.

### Risque 2 — Produire trop de contenus à maintenir

Le second risque est de créer une dette éditoriale trop importante en multipliant les versions de contenus.

Pour éviter ce risque, le site doit privilégier :

- des repères de lecture courts ;
- des résumés structurés ;
- des définitions contextualisées ;
- des blocs réutilisables ;
- des aides à la compréhension ;
- une gouvernance éditoriale claire ;
- une distinction stricte entre texte officiel et contenu d’aide à la lecture.

La logique n’est pas de réécrire chaque décision en plusieurs versions longues, mais de proposer une médiation utile, sobre et maintenable.

### Risque 3 — Confusion entre contenu pédagogique et texte officiel

Les contenus d’aide à la compréhension ne doivent jamais être perçus comme faisant foi.

Chaque page concernée doit rappeler clairement que seul le texte officiel constitue la référence.

### Risque 4 — Architecture trop complexe

La richesse des contenus peut conduire à une architecture trop profonde ou trop segmentée par publics.

Pour éviter ce risque, la navigation principale doit rester courte, testée et orientée vers les intentions principales.

### Risque 5 — UX difficile à réaliser par le lot 2

Les propositions UX/UI doivent être suffisamment précises, documentées et réalistes pour être reprises par le titulaire du lot 2.

Le lot 1 doit donc produire un référentiel exploitable, comprenant :

- parcours ;
- arborescence ;
- wireframes ;
- maquettes ;
- règles d’interaction ;
- composants ;
- design system ;
- spécifications fonctionnelles ;
- règles éditoriales ;
- exigences d’accessibilité ;
- principes responsive ;
- recommandations SEO et écoconception.

---

## 9. Comportements utilisateurs recherchés

Le site réussit si les comportements suivants sont observés :

- un citoyen lit une explication simple puis consulte le texte officiel ;
- un utilisateur utilise la recherche guidée avant d’activer les filtres experts ;
- un juriste active les outils experts sans quitter son parcours ;
- un journaliste identifie rapidement le sens, le statut et les références d’une décision ;
- un utilisateur consulte des décisions liées ou des contenus associés pour approfondir ;
- un utilisateur revient sur le site comme source de référence fiable.

---

## 10. Formule stratégique

La valeur du projet peut être résumée ainsi :

> **Comprendre d’abord. Approfondir sans se perdre. Toujours revenir au texte officiel.**

---

## 11. Définition du succès

La proposition de valeur est validée si le futur site permet de :

- améliorer la compréhension citoyenne ;
- accélérer l’accès à l’information ;
- maintenir la rigueur juridique ;
- renforcer l’efficacité des usages experts ;
- réduire la complexité apparente ;
- limiter la dette éditoriale ;
- produire des livrables directement exploitables par le lot 2 ;
- démontrer les améliorations par des tests utilisateurs.
