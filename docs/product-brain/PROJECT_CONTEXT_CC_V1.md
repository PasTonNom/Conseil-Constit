# Cerveau Produit — Conseil constitutionnel — Lot 1

## Project Context — Prototype d’avant-vente

---

## 1. Contexte

Le projet concerne la réponse au lot 1 de la refonte du site internet du Conseil constitutionnel.

Le lot 1 porte sur :

- la conception ;
- le web design ;
- l’approche UX/UI ;
- l’architecture de l’information ;
- la formalisation des parcours ;
- la production de maquettes ;
- la définition du design system ;
- la production d’un cadre de conception exploitable par le titulaire du lot 2.

L’objectif du cerveau produit est de cadrer un prototype d’avant-vente destiné à accompagner l’offre, à démontrer la compréhension des enjeux et à rendre tangible la proposition UX/UI.

Le prototype n’est pas le produit final.

Il est une preuve de :

- vision ;
- méthode ;
- capacité de conception ;
- compréhension institutionnelle ;
- maîtrise des risques ;
- capacité de transmission au lot 2.

---

## 2. Slogan stratégique

> **Un site citoyen en entrée, expert en profondeur.**

---

## 3. Vision

Le futur site doit permettre à différents profils de comprendre les textes, décisions et contenus du Conseil constitutionnel, tout en leur donnant envie d’approfondir leurs recherches.

L’expérience doit être pensée pour le citoyen par défaut, tout en donnant aux experts un accès rapide aux outils professionnels.

---

## 4. Promesse

> **Comprendre d’abord. Approfondir ensuite. Toujours revenir au texte officiel.**

---

## 5. Publics prioritaires

Les publics prioritaires sont :

1. grand public / citoyens ;
2. experts juridiques ;
3. médias ;
4. enseignants et élèves.

---

## 6. Principe d’expérience

Le site doit proposer une expérience progressive :

- recherche guidée ;
- compréhension simple ;
- accès au texte officiel ;
- lecture expert ;
- outils experts ;
- contenus liés.

Le texte officiel reste toujours la source de vérité.

---

## 7. Pages prioritaires du prototype

Le prototype d’avant-vente doit prioritairement montrer :

1. home ;
2. recherche / liste de décisions ;
3. page décision ;
4. accès expert / recherche avancée.

---

## 8. Pages secondaires à évoquer en intention

Les pages secondaires peuvent être évoquées en intention :

- page Constitution ;
- page actualité / communiqué ;
- passerelle vers “Découvrons notre Constitution”.

---

## 9. Moment signature

Le moment signature du prototype est :

> **Le passage du mode citoyen au mode expert.**

Ce passage doit prouver que le site peut rendre une décision compréhensible sans retirer la profondeur nécessaire aux juristes.

---

## 10. Rôle de “Découvrons notre Constitution”

Le site **“Découvrons notre Constitution”** n’est pas fusionné au futur site principal dans l’hypothèse du prototype d’avant-vente.

Il reste autonome.

Le futur site doit proposer :

- un lien visible ;
- un accès éditorial depuis la home ;
- des passerelles depuis les pages liées à la Constitution ;
- une orientation claire pour les publics pédagogiques.

Cette hypothèse devra être confirmée en phase de cadrage.

---

## 11. IA

L’IA peut être évoquée comme outil interne d’assistance éditoriale et comme capacité d’évolution future.

Elle peut aider à préparer des repères de compréhension à partir du texte officiel.

Ces contenus doivent toujours être :

- relus ;
- corrigés ;
- validés ;
- publiés par les équipes internes du Conseil.

L’IA ne doit jamais être présentée comme :

- une source juridique ;
- un outil de publication autonome ;
- un interprète officiel ;
- une fonctionnalité obligatoire dès le build ;
- une promesse de résumé automatique visible côté utilisateur.

---

## 12. Enseignements issus des questions-réponses client

Les réponses du Conseil constitutionnel confirment plusieurs orientations du cerveau produit.

### Lot 1 et transmission au lot 2

Le lot 1 doit produire un cadre de conception fonctionnelle, UX/UI et architecture de l’information exploitable par le lot 2.

Les spécifications techniques détaillées restent de la responsabilité du lot 2.

### Back-office éditorial

Le lot 1 peut formaliser des principes de contribution, workflows éditoriaux, rôles utilisateurs et gouvernance de contenus.

Il ne doit pas spécifier techniquement le CMS.

### Design system

Le design system attendu relève du référentiel UX/UI, ergonomique et graphique.

Les développements HTML/CSS/JS relèvent du lot 2.

### Direction graphique

Le Conseil constitutionnel n’impose pas le DSFR.

Il attend une proposition graphique cohérente avec l’image institutionnelle du Conseil, en respectant :

- accessibilité ;
- ergonomie ;
- cohérence ;
- sobriété ;
- maintenabilité.

### Pédagogie

Une approche pédagogique renforcée pour les non-experts est attendue.

Des explications structurées peuvent être envisagées à terme.

Le titulaire du lot 1 peut être force de proposition sur la forme, mais pas sur le fond à ce stade.

### Liens entre contenus

Les renvois entre les différents contenus du site peuvent constituer un élément important dans les parcours.

### IA

L’IA doit être abordée comme une exigence d’anticipation et d’évolutivité.

Aucune intégration obligatoire dès le build n’est imposée pour des fonctionnalités de type chatbot, résumé automatique ou moteur de recherche sémantique.

Les cas d’usage peuvent être proposés, à condition d’être qualifiés selon :

- leur valeur ajoutée ;
- leur faisabilité ;
- leurs risques ;
- leur compatibilité avec les exigences institutionnelles.

### Maquettes et gabarits

Le Conseil n’attend pas une maquette pour chaque page du futur site.

Il attend des gabarits, composants, principes d’interaction et maquettes nécessaires à la compréhension de l’expérience utilisateur, du design system et à la bonne réalisation par le lot 2.

---

## 13. Ce que le prototype ne doit pas promettre

Le prototype ne doit pas promettre :

- personnalisation complète par profil ;
- réorganisation dynamique de la home ;
- adaptation automatique des textes ;
- contenus simplifiés pour toutes les décisions dès le lancement ;
- refonte complète de “Découvrons notre Constitution” ;
- IA autonome de résumé juridique ;
- production éditoriale exhaustive ;
- maquette spécifique pour chaque page ;
- développement front-end contractuel côté lot 1.

---

## 14. Style attendu

L’interface doit être :

- républicaine ;
- moderne ;
- claire ;
- sobre ;
- fiable ;
- presque média public ;
- institutionnelle sans être froide ;
- experte sans être illisible.

---

## 15. Mots clés

Les mots clés du projet sont :

- compréhension ;
- référence ;
- accessibilité ;
- sobriété ;
- texte officiel ;

- aide à la compréhension ;

- approfondissement ;
- décision ;

---

## 16. Mots à éviter

Les mots ou formulations à éviter sont :

- vulgarisation ;
- réécriture ;
- disruption ;
- personnalisation automatique ;
- IA générative autonome ;
- simplification excessive ;
- résumé automatique publié ;
- interprétation juridique automatique.

---

## 17. Critères de réussite

Le prototype est réussi si le jury retient :

> **Ils savent concilier citoyens et experts.**

Il doit aussi démontrer :

- une vision claire ;
- une vraie compréhension du Conseil constitutionnel ;
- une proposition UX/UI différenciante ;
- une sobriété institutionnelle ;
- une capacité à valoriser des contenus complexes ;
- une méthode compatible avec le lot 2 ;
- une absence de surpromesse technique ou éditoriale.

## 19. Instruction pour Codex, Figma Make ou Stitch

Lorsqu’un outil IA utilise ce document, il doit respecter les règles suivantes :

- ne pas créer une expérience startup ;
- ne pas créer une personnalisation globale par profil ;
- ne pas faire de l’IA une fonctionnalité visible centrale ;
- ne pas fusionner “Découvrons notre Constitution” ;
- ne pas produire une interface juridique illisible ;
- ne pas masquer les outils experts ;
- ne pas perdre la sobriété institutionnelle ;
- ne pas produire trop de texte dans les interfaces ;
- favoriser une lecture progressive ;
- montrer le passage du mode citoyen au mode expert ;
- concevoir des composants réutilisables ;
- garder une navigation principale courte ;
- produire une interface accessible, claire et maintenable.

---

## 20. Phrase de synthèse

> **Le prototype doit montrer comment le Conseil constitutionnel peut devenir plus lisible pour les citoyens, plus efficace pour les experts, et plus robuste pour sa future réalisation technique.**
